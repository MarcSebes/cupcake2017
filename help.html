To use cupcake, you may need to add a few parameters to the URL. A table of all available options is below.

Some example URLs include:
  • https://cupcake2017.herokuapp.com/?user=MarcSebes
  • https://cupcake2017.herokuapp.com/?user=MarcSebes&date=2017-03-17
  • https://cupcake2017.herokuapp.com/?user=marc&account=eng&date=2017-03-21

Parameter Description Values Required
user The UID of the user account in Validic Any case sensitive value. In the Validic Challenge, it is your First Name and Last Name in Camel Case (e.g. MarcSebes, ShawnSpencer, TerryBolea, VincentFurnier). In the engineering account it typically your lowercase first name Required
date The date of the activity. Note that this is not the updated date for the data. Format is YYYY-MM-DD e.g. 2017-03-17 (Happy St. Patrick's Day) Optional. Defaults to today's date
account The customer account in which this user exists. Valid values are "demo", "dev" (or "devpreview"), and "other". Optional. Defaults to devpreview.
token The developer token for the account. (Note this ultra mega not secure) A guid Required if account="other".
